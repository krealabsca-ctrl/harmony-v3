Harmony v3 · CI/CD Automático

DigitalOcean + GitHub Actions
— Despliegue automático

Cada push a main compila y despliega automáticamente en el Droplet de DigitalOcean.

~30 min setup inicial Ubuntu 22.04 LTS GitHub Actions

Cómo funciona el pipeline

Push a main
→
GitHub Actions
→
SSH al Droplet
→
git pull + build + restart

El pipeline es idéntico en concepto al de Azure: GitHub Actions hace SSH al Droplet y ejecuta un script que actualiza el código, compila y reinicia el servicio.

Diferencias clave vs Azure

Azure vs DigitalOcean — Lo que cambia en esta guía
  • Usuario inicial: el que creaste en la VM
  • IP: se asigna desde el panel del NSG
  • Firewall: Network Security Group en el portal
  • Puerto SSH: por defecto 22 (puede cambiar)
  • Usuario inicial: root
  • IP: visible en el panel del Droplet
  • Firewall: Cloud Firewall en Networking
  • Puerto SSH: siempre 22

Todo lo demás (workflow YAML, deploy.sh, secrets de GitHub) es exactamente igual al manual de Azure CI/CD.

Paso 1 — Preparar el Droplet

Si ya tienes el Droplet con el despliegue manual funcionando, salta al Paso 2. Si es un Droplet nuevo:

# Conectar al Droplet
ssh root@TU_IP_DROPLET

# Actualizar e instalar dependencias
apt update && apt upgrade -y
apt install -y git curl wget build-essential nginx certbot python3-certbot-nginx redis-server

# Go 1.22
wget https://go.dev/dl/go1.22.5.linux-amd64.tar.gz

echo 'export PATH=$PATH:/usr/local/go/bin' > /etc/profile.d/go.sh
source /etc/profile.d/go.sh

# Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# PostgreSQL 16
/usr/share/postgresql-common/pgdg/apt.postgresql.org.sh -y
apt install -y postgresql-16
systemctl enable --now postgresql redis-server

1
Preparar la estructura y clonar el código

useradd --system --no-create-home --shell /bin/false harmony

# Clonar el repositorio

ℹ
Repositorio privado: Ve a tu repo en GitHub → Settings → Deploy Keys → Add deploy key. Pega el contenido de cat ~/.ssh/id_rsa.pub del Droplet (o genera uno con ssh-keygen -t ed25519). Marca Allow write access desactivado (solo lectura).

Paso 4 — Clave SSH para GitHub Actions

3
Generar la clave SSH en el Droplet
# Generar par de claves dedicado para CI/CD
ssh-keygen -t ed25519 -C "github-actions-do" -f /root/.ssh/github_actions_deploy -N ""

# Autorizar la clave pública en este mismo servidor
cat /root/.ssh/github_actions_deploy.pub >> /root/.ssh/authorized_keys
chmod 600 /root/.ssh/authorized_keys

# Mostrar la clave privada — CÓPIALA COMPLETA para el Paso 5
cat /root/.ssh/github_actions_deploy
# -----BEGIN OPENSSH PRIVATE KEY-----
# b3BlbnNzaC1rZXktdjEAAAA...    ← todo este bloque va en el Secret de GitHub
# -----END OPENSSH PRIVATE KEY-----
✗
La clave privada solo va en los Secrets de GitHub. Nunca la guardes en el repositorio ni en ningún archivo de texto.

Paso 5 — Secrets en GitHub

4
Agregar los 3 secrets en el repositorio

Ve a tu repositorio → Settings → Secrets and variables → Actions → New repository secret.

Secret NameValor a pegar
SERVER_HOSTLa IP pública de tu Droplet (ej: 167.99.x.x). La encuentras en el panel de DigitalOcean bajo el Droplet.
SERVER_USERroot — en DigitalOcean el primer acceso siempre es como root.
SSH_PRIVATE_KEYCopia el contenido completo del archivo /root/.ssh/github_actions_deploy, incluyendo las líneas -----BEGIN... y -----END...

Paso 6 — Archivo del workflow de GitHub Actions

5
Crear .github/workflows/deploy.yml en tu repositorio local
.github/workflows/deploy.yml
name: Deploy Harmony v3

on:
  push:
    branches:
      - main

jobs:
  deploy:
    name: Deploy to DigitalOcean Droplet
    runs-on: ubuntu-latest
    timeout-minutes: 15

    steps:
      - name: Deploy via SSH
        uses: appleboy/ssh-action@v1.0.3
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USER }}
          key: ${{ secrets.SSH_PRIVATE_KEY }}
          script: |
            bash /opt/harmony/deploy.sh 2>&1
ℹ
Este workflow es idéntico al de Azure. Los secrets tienen el mismo nombre en ambos. Si usas Azure o DigitalOcean, el .yml no cambia.

Paso 7 — Script de despliegue en el Droplet

6
Crear el script /opt/harmony/deploy.sh en el servidor
nano /opt/harmony/deploy.sh
/opt/harmony/deploy.sh
#!/bin/bash
set -e

export PATH=$PATH:/usr/local/go/bin
REPO_DIR="/opt/harmony/repo"
BIN_DIR="/opt/harmony/bin"
WEB_DIR="/var/www/harmony"

echo "=== [1/5] Actualizando código ==="
cd "$REPO_DIR"
git pull origin main

echo "=== [2/5] Compilando API Go ==="
cd "$REPO_DIR/api"
go mod download
go build -ldflags="-s -w" -o "$BIN_DIR/harmony-api-new" ./cmd/server/main.go
mv "$BIN_DIR/harmony-api-new" "$BIN_DIR/harmony-api"
chown harmony:harmony "$BIN_DIR/harmony-api"

echo "=== [3/5] Compilando Frontend React ==="
cd "$REPO_DIR/web"
npm ci --prefer-offline
npm run build

echo "=== [4/5] Actualizando archivos estáticos ==="
rm -rf "$WEB_DIR"/*
cp -r dist/* "$WEB_DIR/"
chown -R www-data:www-data "$WEB_DIR"

echo "=== [5/5] Reiniciando servicio ==="
systemctl restart harmony-api
systemctl status harmony-api --no-pager

echo "=== Deploy completado: $(date) ==="
chmod +x /opt/harmony/deploy.sh

Paso 8 — Nginx + SSL en el Droplet

7
Configurar Nginx como proxy reverso
nano /etc/nginx/sites-available/harmony
server {
    listen 80;
    server_name harmony.tuempresa.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name harmony.tuempresa.com;

    ssl_certificate /etc/letsencrypt/live/harmony.tuempresa.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/harmony.tuempresa.com/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;

    root /var/www/harmony;
    index index.html;

    location / { try_files $uri $uri/ /index.html; }

    location ~* \.(js|css|png|jpg|ico|svg|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
        client_max_body_size 50M;
    }

    location /ws {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 86400s;
    }
}
ln -s /etc/nginx/sites-available/harmony /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default
nginx -t && systemctl start nginx

# SSL con Let's Encrypt
certbot --nginx -d harmony.tuempresa.com

# Servicio systemd
nano /etc/systemd/system/harmony-api.service
[Unit]
Description=Harmony v3 API
After=network.target postgresql.service redis.service

[Service]
Type=simple
User=harmony
Group=harmony
WorkingDirectory=/opt/harmony
EnvironmentFile=/opt/harmony/.env
ExecStart=/opt/harmony/bin/harmony-api
Restart=always
RestartSec=5
NoNewPrivileges=true

[Install]
WantedBy=multi-user.target
systemctl daemon-reload
systemctl enable harmony-api

Paso 10 — Probar el pipeline

9
Hacer commit del workflow y verificar
# En tu máquina local
git add .github/workflows/deploy.yml
git commit -m "ci: deploy automático en DigitalOcean con GitHub Actions"
git push origin main

Ve a tu repositorio → Actions. El workflow debería aparecer corriendo. Clic en él para ver los pasos en tiempo real.

✓
El deploy demora ~2-3 minutos. Cuando ves la marca ✓ verde en GitHub Actions, el código nuevo ya está en producción en tu Droplet.
⚠
Si falla con "permission denied": El usuario root necesita tener /root/.ssh/authorized_keys con la clave pública correcta. Verifica con: cat /root/.ssh/authorized_keys

Rollback de emergencia

# Opción 1: Revertir en Git → el pipeline se activa con el código anterior
git revert HEAD
git push origin main

# Opción 2: Rollback manual directo en el Droplet
ssh root@TU_IP_DROPLET
cd /opt/harmony/repo
git log --oneline -5               # ver los últimos 5 commits
git checkout HASH_DEL_COMMIT_BUENO  # volver al commit anterior
bash /opt/harmony/deploy.sh         # recompilar y reiniciar
