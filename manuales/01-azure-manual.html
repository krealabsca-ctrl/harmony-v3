Harmony v3 · Guía de Despliegue

Implementación en Microsoft Azure
— Paso a Paso

Go 1.22 + React 19 + PostgreSQL 16 + Redis + Nginx en una VM Ubuntu 22.04 LTS.

~60 min Ubuntu 22.04 LTS Standard_B2ms (2 vCPU / 8 GB)

Prerrequisitos

Antes de comenzar necesitas tener listo lo siguiente:

RequisitoDetalle
Cuenta Azure activaCon acceso al portal portal.azure.com
Dominio o subdominioPor ejemplo harmony.tuempresa.com — necesario para SSL
Código de Harmony v3En un repositorio Git (GitHub/GitLab/Azure DevOps) o archivo ZIP
Claves de APIAnthropic API Key, OpenAI API Key (para el módulo de publicidad)
Cliente SSHTerminal en macOS/Linux, PuTTY o Windows Terminal en Windows

Paso 1 — Crear la Máquina Virtual

1
Ir al portal de Azure y crear un recurso

Entra a portal.azure.com → clic en + Crear un recurso → busca Máquina virtual → clic en Crear.

2
Configurar los parámetros básicos
CampoValor recomendado
Grupo de recursosCrea uno nuevo: rg-harmony-prod
Nombre de la VMvm-harmony-prod
RegiónEast US 2 (o la más cercana a tus usuarios)
ImagenUbuntu Server 22.04 LTS - Gen2
TamañoStandard_B2ms (2 vCPU, 8 GB RAM) — mínimo recomendado
AutenticaciónClave pública SSH (más seguro que contraseña)
Nombre de usuarioazureuser
Par de claves SSHGenera una nueva o sube tu clave pública existente
Disco OSSSD Premium, 64 GB mínimo
✓
Guarda el archivo .pem Cuando Azure genere el par de claves, descargará un archivo .pem. Guárdalo en un lugar seguro — lo necesitarás para cada conexión SSH.
3
Configurar disco y redes

En la pestaña Redes, deja la red virtual predeterminada. En Dirección IP pública selecciona Crear nueva de tipo Estática — esto garantiza que la IP no cambie al reiniciar la VM.

Clic en Revisar y crear → Crear. La VM tarda 2–3 minutos en desplegarse.

Paso 2 — Abrir puertos (NSG)

Por defecto Azure solo abre el puerto 22. Necesitas abrir 80 y 443.

4
Agregar reglas de entrada al Grupo de Seguridad de Red

Ve a tu VM → Configuración → Redes → clic en el nombre del NSG → Reglas de seguridad de entrada → + Agregar.

Agrega estas dos reglas:

NombrePuertoProtocoloPrioridad
Allow-HTTP80TCP300
Allow-HTTPS443TCP310
⚠
No abras el puerto 8080 El API de Go corre en el puerto 8080 internamente. Nginx actúa como proxy. Exponer 8080 directamente es un riesgo de seguridad.

Paso 3 — Conectar por SSH

5
Obtener la IP pública y conectar

En el panel de tu VM verás la Dirección IP pública. Cópiala. Luego en tu terminal local:

# Ajusta los permisos del archivo .pem (obligatorio en macOS/Linux)
chmod 400 ~/Descargas/vm-harmony-prod.pem

# Conectar (reemplaza 20.x.x.x con tu IP pública)
ssh -i ~/Descargas/vm-harmony-prod.pem azureuser@20.x.x.x
✓
La primera vez que te conectas, el sistema te pregunta si confías en el host. Escribe yes y presiona Enter.

Paso 4 — Actualizar el sistema

6
Actualizar paquetes de Ubuntu
sudo apt update && sudo apt upgrade -y
sudo apt install -y git curl wget unzip build-essential

Paso 5 — Instalar Go 1.22

7
Descargar e instalar Go 1.22
# Descargar Go 1.22.5
wget https://go.dev/dl/go1.22.5.linux-amd64.tar.gz

# Extraer en /usr/local
sudo rm -rf /usr/local/go
sudo tar -C /usr/local -xzf go1.22.5.linux-amd64.tar.gz
rm go1.22.5.linux-amd64.tar.gz

# Agregar Go al PATH
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.profile
source ~/.profile

# Verificar la instalación
go version
# Debe mostrar: go version go1.22.5 linux/amd64

Paso 6 — Instalar Node.js 20

8
Instalar Node.js 20 LTS vía NodeSource
# Agregar repositorio oficial de Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -

# Instalar Node.js y npm
sudo apt install -y nodejs

# Verificar
node -v   # v20.x.x
npm -v    # 10.x.x

Paso 7 — Instalar PostgreSQL 16

9
Instalar PostgreSQL 16 desde el repositorio oficial
# Agregar repositorio de PostgreSQL
sudo apt install -y postgresql-common
sudo /usr/share/postgresql-common/pgdg/apt.postgresql.org.sh -y

# Instalar PostgreSQL 16
sudo apt install -y postgresql-16

# Iniciar y habilitar al arranque
sudo systemctl enable --now postgresql

# Verificar que esté corriendo
sudo systemctl status postgresql

Paso 8 — Instalar Redis

10
Instalar Redis 7
sudo apt install -y redis-server

# Configurar Redis para que arranque con el sistema
sudo systemctl enable --now redis-server

# Verificar
redis-cli ping
# Debe responder: PONG

Paso 9 — Instalar Nginx

11
Instalar Nginx y Certbot
sudo apt install -y nginx certbot python3-certbot-nginx
sudo systemctl enable nginx

Paso 10 — Configurar PostgreSQL

12
Crear usuario y base de datos del sistema
# Entrar como postgres
sudo -u postgres psql

-- Dentro de psql: crear usuario (cambia 'MiContraseñaSegura123!' por una tuya)
CREATE USER harmony WITH PASSWORD 'MiContraseñaSegura123!';
CREATE DATABASE harmony_system OWNER harmony;
GRANT ALL PRIVILEGES ON DATABASE harmony_system TO harmony;

-- Dar permisos para crear bases de datos (necesario para provisionar empresas)
ALTER USER harmony CREATEDB;

-- Salir de psql
\q
⚠
Contraseña segura Usa una contraseña de al menos 20 caracteres con letras, números y símbolos. Cópiala — la necesitarás en el archivo .env.

Paso 11 — Crear usuario del sistema

13
Crear usuario de sistema para el servicio

Por seguridad, el API de Go no debe correr como root ni como tu usuario SSH.

# Crear usuario sin shell de login
sudo useradd --system --no-create-home --shell /bin/false harmony

# Crear directorios del proyecto
sudo mkdir -p /opt/harmony/bin
sudo mkdir -p /var/www/harmony
sudo chown -R harmony:harmony /opt/harmony
sudo chown -R harmony:harmony /var/www/harmony

Paso 12 — Subir el código

14
Clonar el repositorio en el servidor
# Si tu repositorio es privado, primero configura un deploy key o HTTPS token
# Opción A: repositorio público o HTTPS con token
sudo -u harmony git clone https://github.com/TU_ORG/harmony-v3.git /opt/harmony/repo

# Opción B: subir los archivos vía SCP desde tu máquina local
# (ejecuta esto en tu terminal LOCAL, no en el servidor)
scp -i ~/vm-harmony-prod.pem -r ./Harmony-v3/ azureuser@20.x.x.x:/tmp/harmony-v3
# Luego en el servidor:
sudo mv /tmp/harmony-v3 /opt/harmony/repo
sudo chown -R harmony:harmony /opt/harmony/repo

Paso 13 — Variables de entorno

15
Crear el archivo .env de producción
sudo nano /opt/harmony/.env

Pega el siguiente contenido, reemplazando los valores marcados con <...>:

# ── Aplicación ────────────────────────────────────────
APP_ENV=production
PORT=8080
FRONTEND_URL=https://harmony.tuempresa.com

# ── Base de datos ─────────────────────────────────────
DB_HOST=localhost
DB_PORT=5432
DB_USER=harmony
DB_PASS=MiContraseñaSegura123!
DB_SSLMODE=disable

# ── JWT ───────────────────────────────────────────────
# Genera con: openssl rand -hex 32
JWT_SECRET=<pega_aqui_64_caracteres_aleatorios>
JWT_EXPIRY_HOURS=24

# ── IA ────────────────────────────────────────────────
ANTHROPIC_KEY=sk-ant-api03-...

# ── Redis / Cola ─────────────────────────────────────
REDIS_URL=redis://localhost:6379

# ── Uploads ──────────────────────────────────────────
UPLOADS_DIR=/opt/harmony/uploads
# Generar el JWT_SECRET con OpenSSL
openssl rand -hex 32

# Proteger el archivo .env (solo lectura para el owner)
sudo chmod 600 /opt/harmony/.env
sudo chown harmony:harmony /opt/harmony/.env

Paso 14 — Compilar el API (Go)

16
Compilar el binario de producción
cd /opt/harmony/repo/api

# Descargar dependencias
go mod download

# Compilar (el flag -ldflags reduce el tamaño del binario)
go build -ldflags="-s -w" -o /opt/harmony/bin/harmony-api ./cmd/server/main.go

# Verificar que el binario existe
ls -lh /opt/harmony/bin/harmony-api
# Debe mostrar algo como: -rwxr-xr-x 1 azureuser azureuser 18M ...

# Dar permisos correctos
sudo chown harmony:harmony /opt/harmony/bin/harmony-api
sudo chmod 755 /opt/harmony/bin/harmony-api

Paso 15 — Compilar el Frontend (React)

17
Construir los archivos estáticos de React
cd /opt/harmony/repo/web

# Instalar dependencias npm
npm ci

# Crear archivo de variables de entorno para el build
cat > .env.production << 'EOF'
VITE_API_URL=https://harmony.tuempresa.com/api
VITE_WS_URL=wss://harmony.tuempresa.com/ws
EOF

# Compilar para producción
npm run build

# Copiar los archivos estáticos al directorio de Nginx
sudo cp -r dist/* /var/www/harmony/
sudo chown -R www-data:www-data /var/www/harmony
ℹ
El directorio dist/ contiene todos los archivos HTML, JS y CSS minificados. Nginx los servirá directamente sin tocar el API de Go.

Paso 16 — Servicio systemd

18
Crear el servicio systemd para el API
sudo nano /etc/systemd/system/harmony-api.service

Pega este contenido exacto:

[Unit]
Description=Harmony v3 API
After=network.target postgresql.service redis.service
Wants=postgresql.service redis.service

[Service]
Type=simple
User=harmony
Group=harmony
WorkingDirectory=/opt/harmony
EnvironmentFile=/opt/harmony/.env
ExecStart=/opt/harmony/bin/harmony-api
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal
SyslogIdentifier=harmony-api

# Límites de seguridad
NoNewPrivileges=true
ProtectSystem=strict
ReadWritePaths=/opt/harmony/uploads
LimitNOFILE=65536

[Install]
WantedBy=multi-user.target
# Crear directorio de uploads
sudo mkdir -p /opt/harmony/uploads
sudo chown harmony:harmony /opt/harmony/uploads

# Recargar systemd y activar el servicio
sudo systemctl daemon-reload
sudo systemctl enable harmony-api
sudo systemctl start harmony-api

# Verificar que esté corriendo
sudo systemctl status harmony-api
✓
Para ver los logs en tiempo real: sudo journalctl -u harmony-api -f

Paso 17 — Configurar Nginx

19
Crear la configuración de Nginx para Harmony
sudo nano /etc/nginx/sites-available/harmony

Pega esta configuración (reemplaza harmony.tuempresa.com con tu dominio real):

# Redirigir HTTP → HTTPS
server {
    listen 80;
    server_name harmony.tuempresa.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name harmony.tuempresa.com;

    # SSL (Certbot llenará estas rutas)
    ssl_certificate /etc/letsencrypt/live/harmony.tuempresa.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/harmony.tuempresa.com/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

    # Headers de seguridad
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    # Frontend React (archivos estáticos)
    root /var/www/harmony;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # Caché de assets (1 año para JS/CSS/imágenes con hash)
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2|woff)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
        access_log off;
    }

    # API Go (proxy inverso)
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
        proxy_connect_timeout 10s;
        client_max_body_size 50M;
    }

    # WebSocket (para el inbox en tiempo real)
    location /ws {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400s;
        proxy_send_timeout 86400s;
    }
}
# Activar el sitio
sudo ln -s /etc/nginx/sites-available/harmony /etc/nginx/sites-enabled/

# Eliminar el sitio por defecto
sudo rm -f /etc/nginx/sites-enabled/default

# Verificar que la configuración es válida
sudo nginx -t

Paso 18 — Certificado SSL

20
Apuntar el DNS y obtener el certificado SSL
⚠
Primero apunta el DNS En tu proveedor de dominio, crea un registro A que apunte harmony.tuempresa.com a la IP pública de tu VM. Espera 5–10 minutos para que propague antes de ejecutar Certbot.
# Reiniciar Nginx antes de obtener el certificado
sudo systemctl start nginx

# Obtener certificado SSL gratuito de Let's Encrypt
sudo certbot --nginx -d harmony.tuempresa.com

# Certbot te pedirá:
# - Tu email (para notificaciones de expiración)
# - Aceptar los términos (Y)
# - Si compartir email con EFF (N)
# Certbot modificará automáticamente el archivo de Nginx con las rutas SSL

# Renovación automática (ya viene configurada, pero verifica)
sudo certbot renew --dry-run

Paso 19 — Verificar que todo funciona

21
Comprobaciones finales
# 1. Estado de todos los servicios
sudo systemctl status harmony-api postgresql redis nginx

# 2. El API responde correctamente
curl https://harmony.tuempresa.com/api/system-config
# Debe devolver: {"app_name":"Harmony","favicon_url":""}

# 3. Ver logs del API en tiempo real
sudo journalctl -u harmony-api -f

# 4. Ver logs de Nginx
sudo tail -f /var/log/nginx/error.log
✓
¡Listo! Accede a https://harmony.tuempresa.com en el navegador. Deberías ver la pantalla de login de Harmony v3.
ℹ
Primer superadmin Crea el superadmin directamente en la base de datos: sudo -u postgres psql -d harmony_system e inserta un usuario con role='superadmin' y password hasheado con bcrypt.

Comandos de mantenimiento

# Reiniciar el API después de actualizar el binario
sudo systemctl restart harmony-api

# Ver últimas 100 líneas de logs
sudo journalctl -u harmony-api -n 100

# Verificar memoria y CPU del servicio
sudo systemctl show harmony-api --property=MemoryCurrent,CPUUsageNSec

# Backup de la base de datos
sudo -u postgres pg_dump harmony_system > /tmp/backup_system_$(date +%Y%m%d).sql
