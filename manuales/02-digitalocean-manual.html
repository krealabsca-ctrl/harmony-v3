Harmony v3 · Guía de Despliegue

Implementación en DigitalOcean
— Paso a Paso

Go 1.22 + React 19 + PostgreSQL 16 + Redis + Nginx en un Droplet Ubuntu 22.04 LTS.

~55 min Ubuntu 22.04 LTS $24/mes — 2 vCPU / 4 GB RAM

Prerrequisitos

RequisitoDetalle
Cuenta DigitalOceanEn cloud.digitalocean.com — necesitas método de pago
Dominio o subdominioPor ejemplo harmony.tuempresa.com
Código de Harmony v3En repositorio Git o archivo local
Claves de APIAnthropic API Key, OpenAI API Key
Clave SSH localSi no tienes: ssh-keygen -t ed25519 -C "tu@email.com"

Paso 1 — Crear el Droplet

1
Agregar tu clave SSH a DigitalOcean

Antes de crear el Droplet, sube tu clave pública SSH para acceso sin contraseña.

Ve a Settings → Security → SSH Keys → Add SSH Key.

# En tu máquina local: mostrar tu clave pública
cat ~/.ssh/id_ed25519.pub
# Copia todo el texto que empieza con "ssh-ed25519 AAAA..." y pégalo en DO
2
Crear el Droplet con la configuración correcta

En el panel de DO: Create → Droplets.

CampoValor
RegiónNueva York 3, San Francisco 3, o Toronto (más cerca de Costa Rica)
Sistema operativoUbuntu 22.04 (LTS) x64
PlanBasic → CPU con uso regular → $24/mes (2 vCPU, 4 GB, 80 GB SSD)
AutenticaciónSelecciona tu clave SSH que subiste antes
Hostnameharmony-prod
BackupsActivar (recomendado — +20% del costo)
IPv6Activar

Clic en Create Droplet. Tarda ~30 segundos en estar listo.

✓
IP estática en DO Los Droplets de DigitalOcean tienen IP pública fija por defecto — no necesitas configurar nada extra.

Paso 2 — Configurar el Firewall

3
Crear un firewall en DigitalOcean

Ve a Networking → Firewalls → Create Firewall.

Reglas de entrada (Inbound):

TipoProtocoloPuertoFuentes
SSHTCP22All IPv4, All IPv6
HTTPTCP80All IPv4, All IPv6
HTTPSTCP443All IPv4, All IPv6

En Apply to Droplets, selecciona tu Droplet harmony-prod. Clic en Create Firewall.

⚠
No abras el puerto 8080. El API corre internamente y Nginx actúa como proxy.

Paso 3 — Conectar por SSH

4
Primera conexión al Droplet

Copia la IP pública de tu Droplet desde el panel de DO (aparece en la vista de Droplets).

# Conectar al Droplet (reemplaza 159.x.x.x con tu IP)
ssh root@159.x.x.x

# La primera vez te preguntará si confías en el host → escribe "yes"
✓
En DigitalOcean, la conexión inicial es como root (a diferencia de Azure que usa un usuario no-root). Esto es normal.

Paso 4 — Configurar DNS

5
Apuntar tu dominio al Droplet

Tienes dos opciones para gestionar el DNS:

Opción A — Usar los nameservers de DigitalOcean (recomendado si usas DO para todo):
Ve a Networking → Domains → Add Domain. Agrega tu dominio y crea un registro A:

TipoHostnameValor
Aharmony159.x.x.x (tu IP del Droplet)

Opción B — Desde tu proveedor de dominio actual:
Crea un registro A en el panel de tu proveedor (GoDaddy, Namecheap, Cloudflare, etc.) que apunte harmony.tuempresa.com a la IP de tu Droplet.

ℹ
La propagación DNS tarda entre 5 minutos y 2 horas. Puedes verificar con: dig harmony.tuempresa.com +short

Paso 5 — Actualizar el sistema

6
Actualizar Ubuntu y crear usuario no-root
# Actualizar paquetes
apt update && apt upgrade -y
apt install -y git curl wget unzip build-essential ufw

# Crear usuario no-root para operaciones del sistema
adduser --disabled-password --gecos "" deploy
usermod -aG sudo deploy

# Copiar las claves SSH al nuevo usuario
rsync --archive --chown=deploy:deploy ~/.ssh /home/deploy

# Configurar UFW (firewall del sistema operativo, capa adicional)
ufw allow OpenSSH
ufw allow 80/tcp
ufw allow 443/tcp
ufw --force enable

Paso 6 — Instalar Go 1.22

7
Instalar Go globalmente
wget https://go.dev/dl/go1.22.5.linux-amd64.tar.gz
rm -rf /usr/local/go
tar -C /usr/local -xzf go1.22.5.linux-amd64.tar.gz
rm go1.22.5.linux-amd64.tar.gz

# PATH global para todos los usuarios
echo 'export PATH=$PATH:/usr/local/go/bin' > /etc/profile.d/go.sh
source /etc/profile.d/go.sh

go version
# go version go1.22.5 linux/amd64

Paso 7 — Instalar Node.js 20

8
Instalar Node.js 20 LTS
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs
node -v && npm -v

Paso 8 — Instalar PostgreSQL 16

9
Instalar PostgreSQL 16 desde el repositorio oficial
apt install -y postgresql-common
/usr/share/postgresql-common/pgdg/apt.postgresql.org.sh -y
apt install -y postgresql-16
systemctl enable --now postgresql

Paso 9 — Instalar Redis

10
Instalar y asegurar Redis
apt install -y redis-server

# Asegurar Redis: solo escucha en localhost
sed -i 's/^# bind 127.0.0.1/bind 127.0.0.1/' /etc/redis/redis.conf
systemctl enable --now redis-server

redis-cli ping
# PONG

Paso 10 — Instalar Nginx y Certbot

11
Instalar Nginx y herramientas SSL
apt install -y nginx certbot python3-certbot-nginx
systemctl enable nginx

Paso 11 — Configurar PostgreSQL

12
Crear usuario y base de datos
sudo -u postgres psql << 'EOF'
CREATE USER harmony WITH PASSWORD 'MiContraseñaSegura123!';
CREATE DATABASE harmony_system OWNER harmony;
GRANT ALL PRIVILEGES ON DATABASE harmony_system TO harmony;
ALTER USER harmony CREATEDB;
\q
EOF
⚠
Cambia MiContraseñaSegura123! por una contraseña única y fuerte. Guárdala para usarla en el .env.

Paso 12 — Estructura de directorios

13
Crear usuario del servicio y directorios
useradd --system --no-create-home --shell /bin/false harmony
mkdir -p /opt/harmony/bin /opt/harmony/uploads
mkdir -p /var/www/harmony
chown -R harmony:harmony /opt/harmony
chown -R www-data:www-data /var/www/harmony

Paso 13 — Subir el código

14
Clonar el repositorio
# Opción A: Clonar desde Git (reemplaza la URL)
git clone https://github.com/TU_ORG/harmony-v3.git /opt/harmony/repo
chown -R harmony:harmony /opt/harmony/repo

# Opción B: Subir por SCP desde tu máquina local
# (ejecuta en tu terminal LOCAL)
scp -r ./Harmony-v3/ root@159.x.x.x:/opt/harmony/repo

15
Crear el archivo .env

Paso 15 — Compilar el API

16
Compilar el binario de Go
cd /opt/harmony/repo/api

go mod download

go build -ldflags="-s -w" -o /opt/harmony/bin/harmony-api ./cmd/server/main.go

chown harmony:harmony /opt/harmony/bin/harmony-api
chmod 755 /opt/harmony/bin/harmony-api

# Verificar
ls -lh /opt/harmony/bin/harmony-api

Paso 16 — Compilar el Frontend

17
Build del proyecto React
cd /opt/harmony/repo/web

cat > .env.production << 'EOF'
VITE_API_URL=https://harmony.tuempresa.com/api
VITE_WS_URL=wss://harmony.tuempresa.com/ws
EOF

npm ci
npm run build

cp -r dist/* /var/www/harmony/
chown -R www-data:www-data /var/www/harmony

Paso 17 — Servicio systemd

18
Crear e iniciar el servicio systemd
nano /etc/systemd/system/harmony-api.service
[Unit]
Description=Harmony v3 API
After=network.target postgresql.service redis.service
Wants=postgresql.service redis.service

[Service]
Type=simple
User=harmony
Group=harmony
WorkingDirectory=/opt/harmony
EnvironmentFile=/opt/harmony/.env
ExecStart=/opt/harmony/bin/harmony-api
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal
NoNewPrivileges=true
LimitNOFILE=65536

[Install]
WantedBy=multi-user.target
systemctl daemon-reload
systemctl enable --now harmony-api
systemctl status harmony-api

Paso 18 — Configurar Nginx

19
Configuración completa de Nginx
nano /etc/nginx/sites-available/harmony
server {
    listen 80;
    server_name harmony.tuempresa.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name harmony.tuempresa.com;

    ssl_certificate /etc/letsencrypt/live/harmony.tuempresa.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/harmony.tuempresa.com/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;

    root /var/www/harmony;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2|woff)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
        access_log off;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
        client_max_body_size 50M;
    }

    location /ws {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 86400s;
    }
}
ln -s /etc/nginx/sites-available/harmony /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default
nginx -t

Paso 19 — SSL con Let's Encrypt

20
Obtener el certificado SSL
⚠
Verifica que el DNS ya propagó antes de ejecutar Certbot: dig harmony.tuempresa.com +short debe devolver tu IP.
systemctl start nginx
certbot --nginx -d harmony.tuempresa.com
# Sigue las instrucciones: email, aceptar términos (Y), no compartir (N)

# Reiniciar Nginx con la nueva configuración SSL
systemctl restart nginx

# Verificar renovación automática
certbot renew --dry-run

21
Comprobar que todo funciona
